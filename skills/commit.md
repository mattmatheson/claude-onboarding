Analyze all staged and unstaged changes using git diff and git status.
Write a clear, concise commit message that:
- Summarizes the nature of the changes (new feature, bug fix, refactor, etc.)
- Focuses on WHY the change was made, not just what changed
- Is 1-2 sentences max for the subject line
- Includes a body with details if the change is complex

Stage relevant files and create the commit. Never commit .env files, credentials, or secrets.
